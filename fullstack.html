What is Full Stack Development?

Full Stack Development refers to the development of both frontend (client-side) and backend (server-side) parts of a web application. Full stack development means building every layer of a web application — what users see, the server that processes requests, and the database that stores data. The word "stack" describes the collection of technologies layered on top of each other to form a complete application. Full-stack developers are skilled in both front-end and back-end languages and technologies, as well as web design and database management. Full-stack developers take part in all aspects of software development, from user experience design to server-side database management and understands how all three layers connect and communicate.

The Three Layers are;

  • Frontend (Client Side): Everything visible in the browser. Built with HTML (structure), CSS (style), and JavaScript (behaviour). Frameworks like Bootstrap speed up development. This is the layer this portfolio focuses on.
  • Backend(Server Side): Runs on a remote server, hidden from users. Handles business logic, authentication, security, and sends data to the front end. Built with languages like Node.js, Python, PHP or C#.
  • Database (Data Layer): Stores all persistent data — user accounts, posts, orders. The back end reads and writes to it. Common types: MySQL / PostgreSQL (structured SQL) and MongoDB (flexible NoSQL documents).

Example: When a user logs in, frontend collects data, backend processes it, and database stores it.

How the Layers Communicate?

The front end sends HTTP requests to the back end. The back end processes the request and sends back a response, usually formatted as JSON — a simple text format both sides can read easily. This request-response cycle is the foundation of all modern web communication.

HTTP Method Description
GET Retrieve data from the server

PUT Update an existing resource on the server
DELETE Delete a resource from the server
